# RDBMS

most basic way to run database is EC2

another service  
AWS RDS - Relational database service  
more convenient -> basically all services are already connected  

Running a single instance as in VM has issues  

Issue-1 reliability  
- Durability (solved by backups provided in RDS but you lose data after last backup)  
- Accessibility (during restore system is offline so clients are lost)  

Solution: create 2 databases with same data (Main <-> Secondary with active sync)  

Problem: sync happens over network -> LAG -> possible data loss  

Solution 1: delay transactions until replication confirmation  
Problem: slows down process  

Solution 2: create multiple duplicate databases and pick fastest 2-3  
Explanation: if probability of delay is 0.01%  
with 2 -> 0.01% x 0.01% = 0.0001%  

---

Issue-2 Scalability  

Vertical -> increase size (CPU/RAM/storage)  
Advantage: everything close to CPU so faster  
Problem: limited  

Horizontal -> add more VMs  
Advantage: theoretically unlimited  
Problem: network communication is slower  

SHARDING -> split data across multiple databases  
Example:  
D1: A-M  
D2: N-R  
D3: S-Z  

Problem: uneven distribution (e.g. 98% data in D2)  

Solution: rebalance  
D1: A-O  
D2: P-R  
D3: S-Z
